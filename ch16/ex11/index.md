# 問題16.11 回答

## 接続のみでの同時接続数確認
HTTP リクエストを送らずにTCP接続のみを維持するスクリプトを作成し確認。
```
node ch16/ex11/hold_connections.js
```

## 確認結果と理由
```
connected: 100
 ・・・
connected: 16300
FAILED at 16331 ok= 16330 fail= 1 err= ENOBUFS
```

- 16330 接続までは接続が確立したが、16331 接続目で `ENOBUFS` エラーが発生した。
- 理由: `ENOBUFS` は OS のネットワークバッファやメモリ等の資源不足を示すエラーであり、同一マシンから大量の接続を保持したことで、OSレベルのリソースが枯渇したことが原因と考えられる。
