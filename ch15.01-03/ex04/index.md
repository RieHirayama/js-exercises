## 問題

グローバルオブジェクトを参照する方法を、ブラウザ内、node 内、ブラウザ node 問わずの３種記しなさい。
また、ブラウザと node のグローバルオブジェクトのプロパティやメソッドを比較し、ブラウザ独自のものを 10 程度記しなさい。
最後に、グローバルオブジェクトに undefined が定義されていることを確認し、過去の ES 仕様でどのような問題が発生していたかを記しなさい。

## 回答

### グローバルオブジェクトの参照方法（3種）

- ブラウザ内

```
window       // もっとも一般的（Windowオブジェクト）
self         // Worker でも使えるブラウザ側の別名
globalThis   // どの環境でも動く共通名

```

※Web Worker は、JavaScriptの重い処理をメインスレッド（UIスレッド）とは別のバックグラウンドスレッドで実行できる仕組み。`Worker` は、そのWeb Workerを作成するためのグローバルコンストラクタ関数。

- Node.js 内

```
global       // Node固有の名前
globalThis   // 共通名（ブラウザでもNodeでも使える）

```

- ブラウザ/Node 問わず（環境非依存）

```
globalThis   // ES2020 以降の標準

```

### ブラウザ独自のプロパティ/メソッド

- `document`（DOM（HTML文書）のルート。要素の取得や操作ができる。）
- `HTMLElement` / `NodeList` （DOM クラス群）
- `navigator`（ユーザーエージェント情報やデバイス情報、Service Workerなどの管理）
- `location`（現在のURL情報やページ遷移の制御）
- `history`（ブラウザ履歴の操作）
- `localStorage` / `sessionStorage` (クライアント側の永続・一時ストレージ)
- `indexedDB` (クライアント側のデータベースAPI)
- `getComputedStyle()` / `CSS`（要素の最終的なCSSスタイルの取得やCSSOM操作）
- `screen`（画面サイズや解像度などの情報）
- `Image` / `Audio`（メディア要素のグローバルコンストラクタ）
- `window.open()` (新しいウィンドウやタブのオープン)
- `alert()` / `confirm()` / `prompt()` (ダイアログ表示API)
- `RTCPeerConnection` / `MediaStream` / `RTCStatsReport`(WebRTC関連のAPI)

### `undefined` はグローバルに定義されているか？過去の問題は？

- 確認
  以下コードで確認。

```jsx
"undefined" in globalThis; // true
globalThis.undefined === undefined; // true
Object.getOwnPropertyDescriptor(globalThis, "undefined");
// { value: undefined, writable: false, enumerable: false, configurable: false }
```

- 過去の ES 仕様で発生していた問題

  - 過去（ES3 時代） `undefined` はただのグローバル変数扱いで、上書き可能だった。
  - そのため、悪意/事故で `undefined` が書き換わると比較やガードが壊れる不具合が発生。
  - IIFE で `undefined` をシャドーイングしたり、`void 0` (常にundefinedになる)を使う回避策が必要だった。
  - ES5 以降、`undefined` は 読み取り専用・再定義不可（`writable:false, configurable:false`）となり、問題は解消。
