## 問題 15.1-3.9 🖋️

React, jQuery などの主要なフロントエンドフレームワークを選び、そのフレームワークを使っていればどのように XSS 対策がされるか、また使っていてもどのような XSS の危険が残るか記述しなさい。

## 回答：Reactについて

### Reactの対策になる理由

- React はデフォルトでほとんどの出力を自動エスケープ（Virtual DOM 経由）するため、通常のレンダリングでは XSS のリスクが低い。
- 自動エスケープ(HTMLエンティティ化）とは、ユーザー入力などの文字列に含まれる特殊な記号（例：<, >, &, " など）を、HTMLとして解釈されないように変換すること

### Reactでも残る危険

Reactを使っていても以下などの危険はある。

- `dangerouslySetInnerHTML`

  - ReactでDOMの innerHTML を直接操作するための特別なプロパティ。
  - 外部／ユーザー由来のHTMLをそのまま挿入すると XSS が発生する。
  - 使用する場合は必ずサニタイズ（ユーザー入力や外部データから危険な内容（例：HTMLタグやJavaScriptなど）を除去・無害化する処理）を事前に行う。

- 外部ライブラリが内部で innerHTML を使っている場合
  - React 自体は安全でも、使っているウィジェットやコンポーネントが内部で `innerHTML` を使うと影響を受ける。
